# Лекция 02. Модульность и связность

## Цель

Научиться видеть модуль как единицу ответственности, а не просто как каталог с файлами.

## Ключевые идеи

- Модуль имеет публичный API и скрытые детали реализации.
- Высокая связность внутри модуля обычно полезнее, чем искусственное дробление.
- Слабая связанность между модулями снижает стоимость изменений.
